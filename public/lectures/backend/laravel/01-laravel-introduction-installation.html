Laravel 详细讲义 - Part 1: Laravel 介绍与安装

这是 Laravel 系列讲义的第一部分 (共 17 部分)。

1.1 Laravel 是什么？

Laravel 是一个开源的、免费的、功能强大的 PHP Web 应用框架。它由 Taylor Otwell 创建，遵循 MVC (Model-View-Controller) 设计模式，旨在让 Web 开发变得更轻松、更快速、更有趣。

  • 框架 (Framework): 它不是一个简单的库，而是一整套用于构建 Web 应用的骨架和工具集。它为你提供了项目结构、常用功能（如路由、数据库操作、模板引擎、用户认证等）和最佳实践，让你不必每次都从零开始“造轮子”。
  • MVC 模式: 这是一种常见的软件设计模式，旨在将应用程序的不同职责分离开：
    • Model (模型): 负责处理应用程序的数据逻辑。通常代表数据库中的表，并包含与数据交互的逻辑。
    • View (视图): 负责展示数据给用户，也就是用户界面 (HTML)。
    • Controller (控制器): 负责接收用户的请求，调用模型处理数据，然后选择合适的视图将结果展示给用户。它是模型和视图之间的协调者。
    Laravel 很好地实践了 MVC 模式，有助于代码的组织和解耦。
  • 优雅的语法和开发体验： Laravel 以其简洁、富有表现力的语法和专注于提升开发者幸福感的特性而闻名。

1.2 为什么选择 Laravel？

PHP 框架有很多，为什么 Laravel 如此受欢迎？

  • 开发效率高： 内置了大量开箱即用的功能（如 Artisan 命令行工具、Eloquent ORM、Blade 模板引擎、队列、任务调度、认证、缓存等），大大减少了常见任务的开发时间。
  • 现代化的特性： 积极拥抱现代 PHP 的特性（如 Composer、命名空间、Trait、类型提示、最新的 PHP 版本支持）和 Web 开发趋势。
  • 庞大且活跃的社区： 你可以很容易地找到大量的教程、文档、第三方扩展包 (Packages) 和遇到问题时的帮助。
  • 优秀的文档： Laravel 的官方文档非常清晰、全面、易于理解。
  • 生态系统丰富： 围绕 Laravel 有很多官方和社区维护的工具和服务，如 Sail (开发环境), Valet (macOS 开发环境), Forge (服务器管理), Vapor (Serverless 部署), Nova (后台管理面板), Laracasts (高质量视频教程) 等。
  • 适合各种规模的项目： 从小型网站到大型企业级应用，Laravel 都能胜任。

1.3 环境要求与准备

在安装 Laravel 之前，你需要确保你的电脑上安装了以下软件：

  • PHP： Laravel 对 PHP 版本有要求。请查阅你打算安装的 Laravel 版本的官方文档，了解它支持的最低和推荐 PHP 版本（通常是较新的版本，如 PHP 8.1+）。验证 PHP 版本：php -v。
  • Composer： Laravel 使用 Composer 来管理项目的依赖。确保你已经全局安装了 Composer。验证：composer --version。
  • 数据库： 大多数 Laravel 应用需要数据库。Laravel 支持多种数据库，最常用的是：
    • MySQL (5.7+) 或 MariaDB (10.3+)
    • PostgreSQL (11.0+)
    • SQLite (3.8.8+)
    • SQL Server (2017+)
    你需要先在你的系统上安装好其中一种数据库。使用集成环境包（如 Laragon, XAMPP, MAMP）通常会自带 MySQL/MariaDB。
  • Node.js 和 NPM (或 Yarn)： Laravel 使用 Vite (或其他工具) 来编译 CSS 和 JavaScript 等前端资源。这需要 Node.js 和 NPM (Node 包管理器，通常随 Node.js 一起安装)。从 Node.js官网 下载安装 LTS (长期支持) 版本即可。验证：node -v 和 npm -v。
  • 一些 PHP 扩展： Laravel 通常需要一些常见的 PHP 扩展被启用，例如：BCMath, Ctype, cURL, DOM, Fileinfo, JSON, Mbstring, OpenSSL, PCRE, PDO (及对应数据库驱动如 pdo_mysql), Tokenizer, XML。大多数集成环境或标准的 PHP 安装会包含这些。你可以通过 phpinfo() 检查。

确保满足这些要求是顺利安装和运行 Laravel 的前提。

1.4 安装 Laravel

有几种创建新 Laravel 项目的方式。

方法一：通过 Composer Create-Project (标准方式)

这是最常用、最通用的方式。打开你的终端或命令提示符，切换到你想要存放项目的父目录（例如你的 Sites 或 www 目录），然后运行：

# 将 "your-project-name" 替换为你想要的项目名称
composer create-project laravel/laravel your-project-name

Composer 会下载 Laravel 安装器以及所有核心依赖，并在当前目录下创建一个名为 your-project-name 的新文件夹，里面就是全新的 Laravel 项目代码。

安装过程可能需要一些时间，因为它需要下载不少东西。安装完成后，你需要进入项目目录：

cd your-project-name

方法二：使用 Laravel Sail (基于 Docker，推荐)

Laravel Sail 是官方提供的、基于 Docker 的本地开发环境解决方案。它的巨大优势在于：

  • 环境一致性： 无论你的电脑是 Windows, macOS还是 Linux，Sail 提供的开发环境都是一致的，避免了“在我机器上能跑”的问题。
  • 简化安装： 它为你打包好了 PHP、Web 服务器（Nginx）、数据库（MySQL/PostgreSQL/Redis等可选服务）、Node.js 等所有需要的环境，你本地只需要安装 Docker Desktop 即可。
  • 易于管理： 使用简单的命令 (`sail up`, `sail down`) 启动和停止整个开发环境。

使用 Sail 创建新项目（需要先安装 Docker Desktop）：

官方推荐的方式是在终端运行一个脚本来安装（具体命令可能随 Laravel 版本略有变化，请参考官方文档）：

# Linux/macOS 示例 (Windows 可能需要用 WSL2)
curl -s "https://laravel.build/your-project-name" | bash

进入项目目录
cd your-project-name

首次启动 Sail (会下载并构建 Docker 镜像，需要较长时间)
./vendor/bin/sail up -d # -d 表示在后台运行

之后启动/停止
./vendor/bin/sail up -d
./vendor/bin/sail down

Sail 启动后，你的 Laravel 应用通常可以通过 http://localhost 在浏览器中访问。

在 Sail 环境中执行命令： 所有需要执行的命令（如 php, composer, npm, artisan）都需要通过 Sail 来执行，以确保它们在 Docker 容器内部运行：

./vendor/bin/sail php -v
./vendor/bin/sail composer install
./vendor/bin/sail npm run dev
./vendor/bin/sail artisan migrate

对于需要频繁使用 ./vendor/bin/sail 的情况，可以设置 shell 别名简化输入：alias sail='[ -f sail ] && bash sail || bash vendor/bin/sail' (将其加入你的 .bashrc 或 .zshrc)。之后就可以直接用 sail artisan ...。

如果你是零基础，强烈建议尝试使用 Sail，它可以帮你规避很多本地环境配置的麻烦。

方法三：Laravel Valet (仅限 macOS)

Valet 是一个为 macOS 设计的极简、高速的 Laravel 开发环境。它使用 Nginx 在后台直接服务你的项目。安装和使用相对简单（需要 Homebrew），适合 Mac 用户。详情参考官方 Valet 文档。

方法四：使用 XAMPP/MAMP/Laragon 等

你也可以在你已有的集成环境中运行 Laravel 项目，但通常需要**手动配置** Web 服务器（Apache 或 Nginx）：

  1. 确保 Web 服务器的文档根目录 (Document Root) 指向 Laravel 项目的 public 目录！这是 Laravel 应用的唯一入口点。
  2. 确保开启了 URL 重写功能（Apache 的 mod_rewrite 或 Nginx 的 try_files 指令），并配置好相应的规则（Laravel 的 public/.htaccess 文件为 Apache 提供了规则，Nginx 需要在虚拟主机配置中添加类似规则）。

手动配置 Web 服务器对于新手来说容易出错，这也是推荐使用 Sail 或 Valet 的原因之一。

1.5 项目目录结构概览

当你创建了一个 Laravel 项目后，会看到很多文件和文件夹。不必立即理解所有内容，先熟悉一下主要目录的用途：

  • app/: 包含应用程序的核心代码。
    • Http/Controllers/: 存放控制器类。
    • Models/: 存放 Eloquent 模型类。
    • Providers/: 存放服务提供者 (Service Providers)。
    • Console/Commands/: 存放自定义的 Artisan 命令。
    • Exceptions/Handler.php: 自定义异常处理逻辑。
    • Http/Kernel.php: 定义全局 HTTP 中间件和路由中间件组。
    • ...
  • bootstrap/: 包含框架启动文件和自动加载配置，以及用于性能优化的缓存文件目录 cache/。
  • config/: 包含所有的应用程序配置文件（数据库、缓存、邮件、队列、服务等）。这些文件返回 PHP 数组，非常易读。
  • database/: 包含数据库迁移 (migrations/)、模型工厂 (factories/) 和数据填充 (seeders/) 文件。
  • public/: Web 服务器的文档根目录。包含入口文件 index.php，以及编译后的前端资源 (CSS, JS)、图片、robots.txt 等需要公开访问的文件。你的 Web 服务器配置应该指向这个目录。
  • resources/: 存放“原始”资源文件。
    • css/, js/: (根据选择的前端方案可能不同) 存放未编译的 CSS 和 JavaScript 源码。
    • lang/: 存放语言文件，用于多语言支持。
    • views/: 存放 Blade 模板（HTML 视图）文件。
  • routes/: 存放所有的路由定义文件。
    • web.php: 定义 Web 界面的路由（应用 Session, CSRF 防护等）。
    • api.php: 定义 API 路由（通常是无状态的，应用 API 中间件）。
    • console.php: 定义基于闭包的 Artisan 命令。
    • channels.php: 定义事件广播频道。
  • storage/: 存放编译后的 Blade 模板、基于文件的 Session、文件缓存、日志文件 (logs/laravel.log)、以及应用生成的文件。
    • app/public/: 存放用户上传的、需要公开访问的文件。你需要运行 php artisan storage:link 来创建一个从 public/storage 到此目录的符号链接。
    • Web 服务器需要对 storage 和 bootstrap/cache 目录拥有写入权限。
  • tests/: 存放自动化测试文件（Feature 功能测试, Unit 单元测试）。
  • vendor/: 存放所有通过 Composer 安装的第三方依赖包。永远不要直接修改这个目录里的任何文件！
  • .env: (需要从 .env.example 复制创建) 环境配置文件，包含数据库密码、API 密钥等敏感信息。不应提交到版本控制。
  • .env.example: .env 文件的模板，列出了需要的环境变量。应提交到版本控制。
  • artisan: Laravel 的命令行接口工具脚本。
  • composer.json / composer.lock: Composer 配置文件和锁定文件。
  • package.json / package-lock.json (或 yarn.lock): NPM (或 Yarn) 的配置文件和锁定文件，用于前端依赖管理。
  • vite.config.js: (或类似文件如 webpack.mix.js 在旧版本) 前端资源编译工具的配置文件。

熟悉这个结构有助于你快速定位代码和配置文件。

1.6 核心配置: .env 文件

Laravel 的配置主要通过 config/ 目录下的 PHP 文件进行管理。然而，为了方便地区分不同环境（开发、测试、生产）的配置，并避免将敏感信息（如数据库密码、API 密钥）直接写入代码或版本控制，Laravel 广泛使用 .env (Environment) 文件。

  • 当你安装完 Laravel 项目后（通过 create-project），根目录下会有一个 .env.example 文件。你需要**复制**它并重命名为 .env。 
    # 在项目根目录下
cp .env.example .env
    如果是克隆现有项目，通常也需要执行此操作。
  • .env 文件是一个简单的键值对文本文件，格式为 KEY=VALUE。
  • Laravel 启动时会加载 .env 文件，并将其中定义的环境变量注入到 PHP 的 _ENV/_SERVER 超全局变量中。
  • 你可以使用辅助函数 env('KEY_NAME', 'default_value') 来获取环境变量的值，如果环境变量未设置，则返回第二个参数指定的默认值。
  • config/ 目录下的配置文件大量使用了 env() 函数来读取 .env 文件中的设置。例如，config/database.php 中可能会有： 
    'mysql' => [
// ...
'host' => env('DB_HOST', '127.0.0.1'),
'port' => env('DB_PORT', '3306'),
'database' => env('DB_DATABASE', 'laravel'),
'username' => env('DB_USERNAME', 'root'),
'password' => env('DB_PASSWORD', ''),
// ...
],
    这意味着数据库连接信息优先从 .env 文件读取，如果 .env 中没有对应项，则使用默认值。
  • 安全：.env 文件包含了敏感信息，永远不要将其提交到 Git 等版本控制系统中！ Laravel 的 .gitignore 文件默认已经包含了 .env。

常见的 .env 配置项：

  • APP_NAME: 应用程序名称。
  • APP_ENV: 当前环境（如 local, testing, production）。
  • APP_KEY: 应用程序密钥（用于加密）。必须设置！
  • APP_DEBUG: 是否开启调试模式（显示详细错误信息）。生产环境必须设为 false！
  • APP_URL: 应用程序的根 URL。
  • LOG</3>_CHANNEL: 默认的日志通道。
  • DB_CONNECTION: 默认数据库连接类型（如 mysql, pgsql, sqlite）。
  • DB_HOST, DB_PORT, DB_DATABASE, DB_USERNAME, DB_PASSWORD: 数据库连接信息。
  • BROADCAST_DRIVER, CACHE_DRIVER, QUEUE_CONNECTION, SESSION_DRIVER, SESSION_LIFETIME: 其他服务的驱动和配置。
  • MAIL_MAILER, MAIL_HOST, MAIL_PORT, etc.: 邮件发送配置。
  • 第三方服务的 API Keys 等。
 

应用密钥 (Application Key)

APP_KEY 是一个非常重要的安全设置。它是一个 32 位的随机字符串，Laravel 使用它来进行加密（例如加密 Session 和 Cookie 数据）。

  • 通过 composer create-project 或 Sail 安装时，通常会自动生成 APP_KEY 并写入 .env 文件。
  • 如果你的 .env 文件中没有 APP_KEY，或者你需要重新生成，可以运行 Artisan 命令： 
    php artisan key:generate
    或者在 Sail 环境中： 
    ./vendor/bin/sail artisan key:generate
  • 必须确保你的 .env 文件中有这个值，否则加密相关的部分将无法工作。

配置缓存

为了提高性能，特别是在生产环境中，Laravel 允许你将所有 config/*.php 文件合并缓存到一个单独的文件中，避免每次请求都去加载和解析多个配置文件。

  • 生成缓存： php artisan config:cache (会创建 bootstrap/cache/config.php)
  • 清除缓存： php artisan config:clear (会删除缓存文件)

注意： 一旦运行了 config:cache，你的应用将只读取缓存文件，不再读取 .env 文件或 config/ 目录下的文件。因此，如果在部署后需要修改配置，你需要先运行 config:clear，修改 .env 文件，然后重新运行 config:cache。

在本地开发过程中，通常不建议运行 config:cache，以便 .env 的修改能立即生效。

1.7 运行你的第一个 Laravel 应用

安装并配置好 .env 文件后，有几种方式可以在本地运行你的 Laravel 应用：

  • php artisan serve:
    • 在项目根目录下运行此命令，会启动 PHP 内建的开发服务器。
    • 通常监听在 [http://127.0.0.1:8000](http://127.0.0.1:8000)。
    • 简单方便，适合快速本地测试，但性能不如 Nginx/Apache，不适合生产环境。
  • Laravel Sail:
    • 如果使用 Sail 安装，运行 ./vendor/bin/sail up -d 启动容器。
    • 应用通常可以通过 http://localhost (或其他配置的端口) 访问。
    • 这是推荐的本地开发方式之一，因为它提供了完整的、隔离的环境。
  • Laravel Valet (macOS):
    • 配置好 Valet 后，它会自动为你项目目录创建一个 *.test 域名。
    • 例如，项目在 ~/Sites/my-project，则可以通过 [http://my-project.test](http://my-project.test) 访问。
    • 非常快速方便，适合 macOS 用户。
  • 其他集成环境 (XAMPP/MAMP/Laragon) 或手动配置：
    • 你需要配置你的 Web 服务器（Apache/Nginx）创建一个虚拟主机 (Virtual Host)。
    • 将虚拟主机的文档根目录 (DocumentRoot / root) 指向 Laravel 项目的 public/ 目录。
    • 确保启用了 URL 重写 (mod_rewrite / try_files)。
    • 然后通过你为虚拟主机配置的域名（例如 [http://laravel.local](http://laravel.local)）来访问。

无论哪种方式，当你成功访问应用的根 URL 时，你应该能看到 Laravel 的默认欢迎页面。
